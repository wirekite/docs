### Mapping

The following table explains how the PostgreSQL datatypes are mapped to the target databases.

| PostgreSQL           | Snowflake       |
| :------------------- | :-------------- |
| **Numbers**          |                 |
| `SMALLINT`           | `NUMBER`        |
| `INT`                | `INT`           |
| `BIGINT`             | `BIGINT`        |
| `NUMERIC`            | `NUMBER`        |
|`FLOAT`               | `FLOAT`                |
|`DOUBLE PRECISION`|`FLOAT`|
| **Times**            |                 |
| `DATE`               | `DATE`          |
| `TIME`               | `TIME`          |
| `TIMESTAMP`          | `TIMESTAMP_NTZ` |
|`TIMESTAMP WITH TIME ZONE`|`TIMESTAMP_LTZ`|
|`TIME WITH TIME ZONE`|`VARCHAR(32)`|
|`INTERVAL`|`VARCHAR(64)`|
| **Strings**          |                 |
| `CHAR`               | `VARCHAR`       |
| `VARCHAR`            | `VARCHAR`       |
| `BPCHAR`            | `VARCHAR`       |
| `TEXT`           | `VARCHAR`       |
| **Blobs**            |                 |
| `BIT`                | `BINARY`        |
| `BOOL`                | `BOOLEAN`        |
| `BYTEA`               | `BINARY`        |
| `JSON`               | `VARCHAR`       |
|`XML`|`VARCHAR`|
| **Miscellaneous**             |                 |
|`MONEY`|`NUMBER(16,2)`|
|`CIDR`|`VARCHAR(64)`|
|`INET`|`VARCHAR(64)`|
|`MACADDR`|`VARCHAR(64)`|
|`UUID`|`VARCHAR(64)`|
|`TSVECTOR`|`VARCHAR(1024)`|
|`TSQUERY`|`VARCHAR(256)`|
|`ARR_<type>`|`VARCHAR(1024)`|
|`POINT`|`VARCHAR(64)`|
|`LINE`|`VARCHAR(64)`|
|`LSEG`|`VARCHAR(64)`|
|`BOX`|`VARCHAR(64)`|
|`PATH`|`VARCHAR(64)`|
|`POLYGON`|`VARCHAR(256)`|
|`CIRCLE`|`VARCHAR(64)`|
|`TXID_SNAPSHOT`|`VARCHAR(64)`|
