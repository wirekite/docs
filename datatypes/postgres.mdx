### Mapping

The following table explains how the PostgreSQL datatypes are mapped to the target databases.

| PostgreSQL           | Snowflake       |
| :------------------- | :-------------- |
| **Numbers**          |                 |
| `TINYINT`            | `TINYINT`       |
| `SMALLINT`           | `NUMBER`        |
| `MEDIUMINT`          | `NUMBER`        |
| `INT`                | `INT`           |
| `BIGINT`             | `BIGINT`        |
| `DECIMAL`            | `NUMBER`        |
| `FLOAT`              | `FLOAT`         |
| `DOUBLE`             | `FLOAT`         |
| **Numbers Unsigned** |                 |
| `TINYINT UNSIGNED`   | `NUMBER`        |
| `SMALLINT UNSIGNED`  | `NUMBER`        |
| `MEDIUMINT UNSIGNED` | `NUMBER`        |
| `INT UNSIGNED`       | `NUMBER`        |
| `BIGINT UNSIGNED`    | `NUMBER`        |
| `DECIMAL UNSIGNED`   | `NUMBER`        |
| `FLOAT UNSIGNED`     | `FLOAT`         |
| `DOUBLE UNSIGNED`    | `N/A`           |
| **Times**            |                 |
| `DATE`               | `DATE`          |
| `TIME`               | `TIME`          |
| `DATETIME`           | `TIMESTAMP_NTZ` |
| `TIMESTAMP`          | `TIMESTAMP_LTZ` |
| `YEAR`               | `INT`           |
| **Strings**          |                 |
| `CHAR`               | `VARCHAR`       |
| `VARCHAR`            | `VARCHAR`       |
| `TINYTEXT`           | `VARCHAR`       |
| `MEDIUMTEXT`         | `VARCHAR`       |
| `TEXT`               | `VARCHAR`       |
| `LONGTEXT`           | `VARCHAR`       |
| **Blobs**            |                 |
| `BIT`                | `BINARY`        |
| `BINARY`             | `BINARY`        |
| `VARBINARY`          | `BINARY`        |
| `TINYBLOB`           | `BINARY`        |
| `MEDIUMBLOB`         | `BINARY`        |
| `BLOB`               | `BINARY`        |
| `JSON`               | `VARCHAR`       |
| **Sets**             |                 |
| `ENUM`               | `VARCHAR(64)`   |
| `SET`                | `VARCHAR(64)`   |
