### Mapping

The following table explains how the Oracle datatypes are mapped to the target databases.

| Oracle Datatype | Firebolt Datatype | Snowflake Datatype |
| --------------- | ----------------- | ------------------ |
| **Numbers** | | |
| `NUMBER` | `NUMERIC` | `NUMBER` |
| `BINARY_FLOAT` | `FLOAT,REAL` | `FLOAT` |
| `BINARY_DOUBLE` | `DOUBLE` | `FLOAT` |
| **Times** | | |
| `DATE` | `TIMESTAMP` | `TIMESTAMP` |
| `TIMESTAMP` | `TIMESTAMP` | `TIMESTAMP` |
| `TIMESTAMP WITH TIME ZONE` | `TIMESTAMPTZ` | `TIMESTAMP_TZ` |
| **Strings** | | |
| `CHAR` | `TEXT` | `VARCHAR` |
| `NCHAR` | `TEXT` | `VARCHAR` |
| `VARCHAR2` | `TEXT` | `VARCHAR` |
| `NVARCHAR2` | `TEXT` | `VARCHAR` |
| `CLOB` | `TEXT` | `VARCHAR` |
| `NCLOB` | `TEXT` | `VARCHAR` |
| `LONG` | `TEXT` | `VARCHAR` |
| `BFILE` | `TEXT` | `VARCHAR` |
| **Blobs** | | |
| `RAW` | `BYTEA` | `BINARY` |
| `LONG RAW` | `BYTEA` | `BINARY` |
| `BLOB` | `BYTEA` | `BINARY` |
