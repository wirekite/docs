### Mapping

The following table explains how the MySQL datatypes are mapped to the target databases.

| MySQL                | PostgreSQL | Snowflake |
| :------------------- | :--------- | :-------- |
| `BIT`                |`BIT`       |           |
|**Numbers**           |            |           |
| `TINYINT`            |`SMALLINT`            |           |
| `SMALLINT`           |`SMALLINT`            |           |
| `MEDIUMINT`          |`INT`            |           |
| `INT`                |`INT`            |           |
| `BIGINT`             | `BIGINT`           |           |
| `DECIMAL`            | `NUMERIC`           |           |
| `FLOAT`              | `FLOAT`           |           |
| `DOUBLE`             | `DOUBLE PRECISION`           |           |
|**Numbers Unsigned**  |            |           |
| `TINYINT UNSIGNED`   | `SMALLINT`           |           |
| `SMALLINT UNSIGNED`  | `INT`          |           |
| `MEDIUMINT UNSIGNED` | `INT`           |           |
| `INT UNSIGNED`       | `BIGINT`           |           |
| `BIGINT UNSIGNED`    | `NUMERIC(21)`          |           |
| `DECIMAL UNSIGNED`   | `NUMERIC(N+1)`           |           |
| `FLOAT UNSIGNED`     | `DOUBLE PRECISION`           |           |
